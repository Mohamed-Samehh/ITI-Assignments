Algorithm Complexity

Bubble Sort

Slow
Best O(n)
AverageO(n²)
Worst O(n²)
Space O(1)

Repeatedly compares adjacent elements and swaps them if out of order. Simple but inefficient for large datasets.

Selection Sort

Slow
Best O(n²)
AverageO(n²)
Worst O(n²)
Space O(1)

Finds the minimum in the unsorted region and places it at the front. Always O(n²) comparisons regardless of input.

Insertion Sort

Medium
Best O(n)
AverageO(n²)
Worst O(n²)
Space O(1)

Builds the sorted array one element at a time by inserting each into its correct position. Very efficient on nearly-sorted data.

Merge Sort

Fast
Best O(n log n)
AverageO(n log n)
Worst O(n log n)
Space O(n)

Divides the array in half, recursively sorts both halves, then merges them. Consistently efficient but requires extra memory.

Quick Sort

Fast
Best O(n log n)
AverageO(n log n)
Worst O(n²)
Space O(log n)

Picks a pivot, partitions elements around it, then recursively sorts each side. Fastest in practice despite an O(n²) worst case.

Heap Sort

Fast
Best O(n log n)
AverageO(n log n)
Worst O(n log n)
Space O(1)

Builds a max-heap then repeatedly extracts the maximum. Guaranteed O(n log n) with no extra memory — the best of both worlds.

Shell Sort

Medium
Best O(n log n)
AverageO(n¹·²⁵)
Worst O(n²)
Space O(1)

An improved Insertion Sort that sorts elements at decreasing gaps, reducing the gap to 1. Much faster on medium-sized arrays.

Counting Sort

Linear
Best O(n + k)
AverageO(n + k)
Worst O(n + k)
Space O(k)

A non-comparison sort that counts element frequencies to directly place each element. Blazing fast when value range k is small.

Radix Sort

Linear
Best O(nk)
AverageO(nk)
Worst O(nk)
Space O(n + k)

Sorts integers digit by digit from least to most significant using Counting Sort as a subroutine. Non-comparison & effectively linear.

Cocktail Sort

Slow
Best O(n)
AverageO(n²)
Worst O(n²)
Space O(1)

Bidirectional Bubble Sort that alternates forward and backward passes, shrinking the active window from both ends simultaneously.

Comb Sort

Medium
Best O(n log n)
AverageO(n² / 2ᵖ)
Worst O(n²)
Space O(1)

Improves Bubble Sort by comparing elements at a shrinking gap (factor 1.3). Eliminates small-value stragglers near the end efficiently.

Gnome Sort

Slow
Best O(n)
AverageO(n²)
Worst O(n²)
Space O(1)

Like a garden gnome sorting flower pots — steps forward until it finds an out-of-order pair, swaps it back, then continues. Charming but slow.

Cycle Sort

Slow
Best O(n²)
AverageO(n²)
Worst O(n²)
Space O(1)

Theoretically optimal for minimising memory writes — each value is written at most once. Ideal for write-expensive storage such as flash memory.

Odd-Even Sort

Slow
Best O(n)
AverageO(n²)
Worst O(n²)
Space O(1)

Alternates between odd-index and even-index adjacent comparisons. Inspired by parallel sorting networks where each phase runs simultaneously.
